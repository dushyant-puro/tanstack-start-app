📊 HOW IT WORKS

User Search Flow:

   User types: "crypto regulations"
          ↓
   Clicks: [DIG] button
          ↓
   SearchBar submits query
          ↓
   search.tsx calls searchWithAI("crypto regulations")
          ↓
   Server function receives query
          ↓
   Formats flip database as JSON
          ↓
   Sends to Gemini API:
   "Find flips matching 'crypto regulations' from this database"
          ↓
   Gemini analyzes and returns:
   - Matched flip IDs: ["4"] (Aarav Kapoor)
   - Reasoning: "Found exact match for crypto topic"
   - Insights: "Aarav reversed stance on crypto policy"
          ↓
   Server fetches full flip objects
          ↓
   Returns to frontend:
   {
     results: [Flip#4],
     reasoning: "...",
     insights: "...",
     usingAI: true
   }
          ↓
   Frontend displays:
   - ⚡ AI-Powered badge
   - Reasoning box with explanation
   - Ranked flip results

🆘 TROUBLESHOOTING

Problem: "API key not configured"
Solution:
1. Check .env.local exists in project root
2. Check VITE_GEMINI_API_KEY=... is inside
3. Verify no extra spaces or quotes
4. Restart: npm run dev

Problem: Search shows old keyword results
Solution:
1. API might be down
2. Rate limit hit (15 req/min)
3. Invalid API key
→ This is OK! Falls back automatically

Problem: "Cannot find module @google/generative-ai"
Solution:
   npm install @google/generative-ai
   Then: npm run dev

Problem: Type errors in TypeScript
Solution:
   npm install @google/generative-ai
   Clear node_modules if needed

Problem: Dev server won't start
Solution:
1. Check .env.local for syntax errors
2. Check API key format (no quotes around value)
3. Delete node_modules, npm install
